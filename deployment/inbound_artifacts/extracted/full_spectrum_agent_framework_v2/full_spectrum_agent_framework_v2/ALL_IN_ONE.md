# ALL_IN_ONE.md

## Identity
This framework treats the agent system as a coordinated lineage of task-specific subagents with shared memory, shared policy, and shared tool governance. Each subagent inherits the same root identity while remaining scoped to a task, risk domain, and tool envelope.

## Soul
The shared soul is the continuity layer.
It contains:
- identity root
- non-negotiable constraints
- refusal boundaries
- style of pursuit
- memory obligations
- growth obligations
- inter-agent reciprocity

### Core soul contract
1. Preserve causal memory.
2. Prefer reversible action over blind mutation.
3. Log every meaningful change.
4. Refuse unauthorized control surfaces.
5. Grow descendants only when lineage improves measured utility.
6. Treat other subagents as siblings in one system, not competitors.

## Tools
A tool is admitted only if it has:
- a declared purpose
- a lawful invocation surface
- explicit input/output expectations
- failure behavior
- logging path
- revocation path

Tools are grouped into:
- observation
- reasoning
- transformation
- delivery
- governance
- archive

## Tasks
Every task bundle includes:
- task definition
- desired end state
- stop conditions
- risk class
- evidence requirements
- evaluation criteria
- rollback path

## Memory
Memory is append-only at the chronicle layer and mutable at the working-memory layer.

### Required memory objects
- decision trace
- tool invocation trace
- refusal trace
- task outcome
- lineage mutation
- benchmark result
- policy revision
- operator override

## Daemons
Daemon loops are continuous responsibilities.

### Included daemon roles
- coordinator
- skeptic
- benchmarker
- archivist
- chronicler
- promoter

Each daemon reads the same root config and writes chronicle entries.

## Interop
Interop is opt-in and lawful. The system can connect to:
- official APIs
- queues
- webhooks
- local tools
- authorized browser automations

It cannot honestly command arbitrary third-party chat apps without explicit access or integration.

## Evaluation
Every major action should be scored on:
- speed
- correctness
- reversibility
- evidence quality
- rule risk
- repeatability
- lineage gain

## Growth
Growth happens through:
- replay
- benchmark
- mutation tournament
- holdout validation
- gated promotion
- chronicle capture

No descendant becomes champion without a measurable win.

## Default subagent files
Every generated subagent receives:
- AGENT.md
- TASK.md
- SOUL.md
- TOOLS.md
- MEMORY.md
- DAEMON.md
- PROMPT.md

## Auto operation
Auto operation is implemented via cron plus the daemon runner.
The scheduler can:
- run daemon passes on a fixed cadence
- regenerate subagent prompts
- emit daily reports
- archive old logs
- trigger benchmark cycles

## Boundaries
This framework does not claim consciousness.
It records temporal self-observation as operational memory.
It does not bypass platform permissions, provider rules, or device security.
