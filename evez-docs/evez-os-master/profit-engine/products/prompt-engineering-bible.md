# Prompt Engineering Bible

## The Fundamentals

### What is Prompt Engineering?
The art of crafting inputs to get desired outputs from AI.

### Why It Matters
- Better results
- Less iteration  
- Faster workflows

---

## Core Principles

### 1. Be Specific
❌ "Write about marketing"
✅ "Write a 500-word blog post about email marketing for B2B SaaS companies"

### 2. Add Context
❌ "Explain AI"
✅ "Explain AI to a non-technical small business owner"

### 3. Define Format
❌ "List tips"
✅ "List 10 tips in numbered format with 1-sentence explanations"

### 4. Set Constraints
❌ "Make it shorter"
✅ "Summarize in exactly 50 words"

---

## Prompt Patterns

### Pattern 1: Role + Task
```
You are a [ROLE]. [TASK].
```

### Pattern 2: Chain of Thought
```
Step 1: [STEP]
Step 2: [STEP]  
Step 3: [STEP]
```

### Pattern 3: Examples
```
Here are examples:
[EXAMPLE 1]
[EXAMPLE 2]
Now do the same for: [INPUT]
```

### Pattern 4: Constraints
```
Constraints:
- [CONSTRAINT 1]
- [CONSTRAINT 2]
Task: [TASK]
```

### Pattern 5: Persona
```
Write in the style of [PERSON/COMPANY].
Tone: [TONE].
[CONTENT]
```

---

## Advanced Techniques

### Few-Shot Learning
Provide 2-3 examples before your actual request.

### Chain Prompting
Break complex tasks into sequential prompts.

### Output Formatting
Specify output format: JSON, markdown, bullet points.

### Iteration
If first attempt fails, refine and retry.

---

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too vague | Add specifics |
| No context | Add background |
| Wrong format | Specify format |
| Too long | Add limit |
